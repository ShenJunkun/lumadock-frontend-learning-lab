# 99 待补充的学习模块

本文档列出当前项目作为前端学习项目尚未覆盖的重要技能，供后续扩展参考。

当前状态：P0、P1、P2、P3、P4、P5、P6 已完成。P5 已补齐 PWA / 离线、Web Vitals / 性能预算、安全与隐私加固、视觉回归、设计系统深化和状态建模。P6 已完成 npm workspace、`apps/web`、`apps/api`、`packages/ui` 和 `packages/api-client`。P7 已完成轻实现：错误监控 adapter、feature flags、发布 runbook、环境策略、真实鉴权策略和数据库迁移/备份演练文档。P8 尚未完成，继续作为下一阶段高级前端专项。P9-P12 作为远期研究方向，覆盖大型应用平台化、产品增长实验、多端扩展和 AI 前端能力。

---

## 下一步推荐路线

P7 已按“轻实现 + 文档”落地：真实 Sentry/Auth/迁移平台暂不接入，但 adapter、feature flag 和 runbook 边界已经清楚。下一步建议进入 P8，优先做 Storybook / 组件文档站，再选择复杂交互或 SSR / Next.js 迁移评估。P9-P12 不作为近期必做主线，只作为项目继续拉长时的研究路线。

## P10：产品增长与实验（远期）

P10 关注“做了功能以后是否真的有效”。它适合在产品体验稳定后再补。

| 模块                   | 要做什么                                           | 验收标准                                      |
| ---------------------- | -------------------------------------------------- | --------------------------------------------- |
| **A/B Test**           | 设计实验分组、实验配置、指标假设和停止条件         | 实验不是随机改 UI，而是有目标、有指标、有复盘 |
| **漏斗分析**           | 建立访问、浏览产品、进入预约、提交成功的转化链路   | 能定位用户在哪一步流失                        |
| **埋点治理**           | 建立事件命名、属性白名单、隐私边界和版本变更规则   | 事件可分析、可维护，并且不采集无关个人信息    |
| **指标看板**           | 把性能、转化、错误、核心行为汇总成产品观察面板     | 看板能回答具体问题，而不是只堆图表            |
| **增长实验复盘**       | 记录假设、改动、结果、结论和下一步动作             | 每次实验都能转化成明确的产品或工程决策        |
| **隐私友好的产品分析** | 学习最小采集、聚合统计、匿名化和用户退出分析的边界 | 增长分析不破坏 P4/P5 已建立的隐私友好原则     |

| ---------------------- | -------------------------------------------------- | ---------------------------------------- |

| **Electron / Tauri**   | 学习桌面端窗口、文件系统、安全沙箱和自动更新       | 能说明桌面端不是简单套壳，并有安全边界   |
| **小程序评估**         | 学习小程序包体积、路由、组件模型、登录和审核流程   | 能判断当前产品是否值得做小程序版本       |
| **跨端设计系统**       | 把 token、组件语义、图标、间距和动效拆成跨端层级   | 共享设计语言，不强行共享所有组件实现     |
| **共享业务模型**       | 抽取产品、预约、权限、配置等跨端可复用的类型和规则 | 不同端复用稳定模型，同时保留端特有交互   |

## P12：AI 前端能力（远期）

P12 只有在项目边界允许接入 AI 能力时才启动。重点不是炫技，而是让用户保持控制权、理解系统行为，并能安全失败。

| 模块                   | 要做什么                                       | 验收标准                                            |
| ---------------------- | ---------------------------------------------- | --------------------------------------------------- |
| **智能搜索**           | 设计语义搜索、普通搜索回退、引用来源和权限过滤 | 搜索结果可解释，可回退，不越权展示数据              |
| **内容生成**           | 为产品文案、配置建议或后台摘要设计生成入口     | AI 输出必须可编辑、可确认，不自动替用户提交关键动作 |
| **Agent UI**           | 设计计划、步骤、工具调用、确认、中断和恢复界面 | 用户能看懂 AI 正在做什么，并能暂停、撤销或改写目标  |
| **流式响应体验**       | 处理 streaming 文本、加载状态、取消请求和重试  | 长任务有过程反馈，失败时能保留上下文                |
| **AI 安全与可解释性**  | 记录数据权限、提示注入、越权操作和输出审核策略 | 有安全边界和失败兜底，不把 AI 当成无条件可信系统    |
| **人机协作工作流评估** | 判断哪些任务适合 AI 辅助，哪些必须保留人工确认 | AI 能减少重复劳动，但不会模糊责任归属和最终确认环节 |

## P12 之后的边界

P12 之后不再继续追加固定编号。新的想法先进入“远期研究 backlog”，只有当它能明确服务当前项目目标、并且不拖慢 P5-P8 主线时，再决定是否升格成新的阶段。
